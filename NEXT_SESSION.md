# План на наступну сесію

Актуально станом на 2026-07-22. Повна історія та деталі виконаного — у `README.md`.

## 1. Вичитка контенту власником (головне, блокує решту)

- [ ] Пройтись по PL та EN сторінках на стейджингу `igraphite.pl/wp-new/`
- [ ] Позначити, де верстка зламана або текст не відповідає дійсності
- [ ] Перевірити переклад 8 нових EN-сторінок каталогу продукції
      (`products-refractories`, `products-carbon-brushes`,
      `products-heat-insulation`, `products-automotive`, `products-friction`,
      `products-formed-graphite-parts`, `products-lubricants`,
      `projects-standard`) — перекладено вручну, машинним перекладачем не
      перевірялось
- [ ] Вирішити, чи перекладати EN-версії двох сторінок каталогу, які поки
      без перекладу: "Grafit żyłowy", і чи потрібен ще якийсь контент

## 2. Дрібний технічний борг (не блокер, можна паралельно)

- [ ] Футер (`footer.php`) на EN-сторінках показує посилання польською —
      треба зробити мовно-залежним (аналогічно до головного меню, яке вже
      розділили на PL/EN)

## 3. Перехід на прод (після вичитки контенту)

- [ ] Перенести WordPress з `/wp-new/` у корінь домену
- [ ] Оновити захардкожені шляхи, залежні від `/wp-new/` (форма контактів
      зараз шле на `/wp-new/wp-admin/admin-post.php`)
- [ ] Заархівувати (не видаляти одразу) старі статичні файли з
      продакшн-кореня
- [ ] Прибрати залишки старого AJAX-шляху форми контактів на живому сайті
      (зараз фолбек `contact.php` без AJAX працює, але основний JS-шлях
      404-ить, бо `index.php` з живими креденшелами вже видалено) — після
      переходу на прод стара форма замінюється новою WP-формою і питання
      закривається само собою
- [ ] Визначити процес деплою з GitHub на хостинг надалі (FTP/SFTP при
      пуші, чи вручну)

## Довідково: де що на сервері

- Стейджинг: `igraphite.pl/wp-new/`, вебрут `~/igraphite.pl/www/wp-new/`
- SSH: алiас `igraphite` (`~/.ssh/config`), PHP 8.5 для WP-CLI —
  `/usr/local/php85/bin/php /usr/local/bin/wp <command>`
- WP DB: `wpotas_igraphite`
